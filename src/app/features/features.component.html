Features

Trackers: Design the flexible data inputs yourself

You can design your own data format by configuring various data types such as number, text, time point, time range, choices, location, or audio records. You can also change the configuration even if you've already been tracking data.

The configuration UI is user-friendly, similar to those of form-building systems such as Google Forms or Survey Monkey.

Services: Integrate external trackers and other services

You can connect the external services to OmniTrack's data field. For example, you can connect Fitbit step count to a number field. When you open the input screen or conduct background logging, OmniTrack automatically load value from the connected service.

Triggers: Automate value retrieval and data logging

You can conduct an automatic tracking using triggers. Once triggers are fired, OmniTrack retrieves tracking values from external services and automatically logs the values.

OmniTrack supports two types of triggers:
Time-based Time-based triggers are fired either at the preset time or at periodic intervals.
Data-driven Data-driven triggers are fired when a value from the external service passes the threshold.
Currently, data-driven trigger is under maintenance due to the battery drain issue. We'll release the feature soon.

Streamline tracking by shortcuts, widgets, and reminders

OmniTrack offers functionalities which facilitate your tracking. By using a shortcut panel on notification list, you can quickly log your data without opening the application. You can either open the input page or log an item instantly with automatically filled values. If you mind your notification panel is occupied by the shortcut panel, consider using OmniTrack's screen widgets instead.
If you made a tracker which you have to keep track of regularly (e.g., a diary), you can setup reminders which notifies you at specific time or at periodic intervals.

Visualization and feedback

You can review your raw data in an item list. If you want to obtain the holistic insights, visit the automatically curated visualization page.

The visualization is automatically curated regarding the tracker's schema; i.e., What types of fields exist and how they are configured in the tracker.
