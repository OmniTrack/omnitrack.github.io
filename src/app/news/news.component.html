OmniTrack Roadmap

2017 December - Open Source with a Standalone Server

We are currently working on reimplementing the OmniTrack backend server from scratch, by removing dependency on Google Firebase. We will open the source code of the OmniTrack Android, and associated server running on Node.js to allow researchers manage the tracker data in their own server storage.

Call for Researchers

We are looking for researchers who are interested in conducting a data collection study by diaries or experience sampling (EMA) using the OmniTrack platform. We are working on a research platform by extending OmniTrack to be used by researchers for conducting data collection studies. As a first step of this, we are designing the researchers' interface which supports diary study.

The example diary studies we envision follows:
  1. Self-report diaries which are personalized for each participant
  2. Self-reports + passively gathered fitness device data
  3. Data collection involving multiple sensing devices
If you are interested in such types of studies and have any inquiries, feel free to contact us!

Important Note

Currently, OmniTrack for Android heavily depends on Google Play Services, including GCM and Firebase. This implies that OmniTrack CANNOT be targeted for the citizens in the countries which are restricting the access to the Google services (China is one prominent region for such restrictions).
Please take this into account when you plan your deployment target audience.
